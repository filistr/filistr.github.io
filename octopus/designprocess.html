Octopus Game

Inspiration & Design Process

The design process for this game started with creating basic visual and movement functions for the octopus. Furthermore, the fishes were created and the collision detection between the fishes and octopus was implemented. Thereafter, the feature of the octopus releasing ink and the collision detection between fish and ink were created.

The detail that a fish might need to collide with ink several times depending on which size it has is inspired from both Bubble Struggle 2 and Age of War. Both these games have features that the game’s obstacle is not defeated immediately, but might need to be attacked several times. In the Octopus game, this kind of feature was implemented through that the fishes shrunk in size each time they collided with ink. When a fish reached a specific size, they were viewed as finally defeated. The feature of the game’s obstacle shrinking in size when getting attacked is also inspired by Bubble Struggle 2 where the bubbles get smaller and smaller each time they are popped.

Furthermore, the feature of the octopus needing to pick up an ink bottle in order to release ink was implemented. After the basic game functionality was implemented, more advanced visual features were introduced such as using a sprite sheet for animation and perlin noise for movement. Lastly, the HTML page with the select difficulty dropdown button, regular buttons, game and control instructions was created.
